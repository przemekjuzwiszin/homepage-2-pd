Index pomaga w codziennej pracy. Dzięki temu mechanizmowi możemy dokładnie doprecyzować, co chcemy zacommitować. Index to przestrzeń pomiędzy working copy a commitem. Tworzymy zmiany, małe, spójne commity, zbliżone odpowiedzialnością. Nie łączymy zmian funkcjonalności naszego systemu ze śmieciowymi zmianami ponieważ utrudniają pracę. Wchodzimy w dwa nowe tryby gita interactive i patch. Poznajemy pojęcie hunk, SRP i TDD. Dzielimy zmiany na osobne, konkretne commity.
